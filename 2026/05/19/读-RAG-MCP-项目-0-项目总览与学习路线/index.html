本系列目标：通过完整复现一个工业级 RAG 项目，把 RAG 面试中的高频考点（混合检索、Rerank、Ragas 评估、MCP 协议）一次性吃透。

我看完 README 后觉得它有三个不同于其他 RAG demo 的特点：

  1. 不是玩具：完整覆盖了 Ingestion → Retrieval → Generation → Evaluation 全链路，不像很多教程只做检索那一步
  2. MCP 协议加成：当下大模型工具调用最热的协议，可直接被 Claude Desktop / Cursor 等客户端调用
  3. 工程化思路：用 DEV_SPEC 驱动开发的方式，本身就是一种值得学习的方法论

二、项目能做什么（一句话版）

一个可插拔的本地知识库问答服务：把 PDF 等文档喂给它，它能用 LLM 帮你回答关于这些文档的问题，并标明引用出处；同时把自己暴露成 MCP 工具，让 Claude / Copilot 直接调用。

四、技术栈速览

层级 选型 我的初印象
语言 Python 3.10+ 标配
向量库 ChromaDB 比 FAISS 重，但带 metadata 过滤
稀疏检索 自实现 BM25 + Jieba 分词 中文场景必备
文本切分 langchain-text-splitters 不重新造轮子
PDF 解析 markitdown[pdf] 微软出的，能保留结构
MCP 协议 mcp 官方 SDK Anthropic 推的协议
Dashboard Streamlit 数据科学家最爱，写得快
评估 Ragas + 自定义 Evaluator 业界主流
